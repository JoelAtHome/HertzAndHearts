# Community Posting Plan (Beta Testers)

Use this as a light posting plan so outreach stays focused and respectful.

## Priority Channels

- Reddit: r/Biohackers, r/HRV, r/QuantifiedSelf, r/opensource
- Hacker News: Show HN
- Quantified Self forum
- Relevant Discord/Slack groups (coaching, biofeedback, wearables)
- LinkedIn and X personal network

## Post Timing

- Wave 1: launch day (Reddit + X/LinkedIn + Show HN)
- Wave 2: 3-4 days later with "early findings + still recruiting testers"
- Wave 3: final call 3 days before beta window closes

## Message Rules

- Lead with scope: "public beta, need checklist-based testers"
- Keep disclaimer visible: research use only; non-diagnostic
- Ask for structured reports, not general comments
- Thank communities and avoid repetitive spam posts

## Suggested CTA

"Run `docs/PUBLIC_RELEASE_CHECKLIST.md` and file results via issue template."

## Beta Intake Targets

- 10-20 total testers
- 3+ operating system variants
- 2+ sensor models
- 20+ checklist-driven reports/issues
