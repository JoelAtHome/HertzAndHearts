VNS-TA Developer Mode

Quick Reference Guide — For development and testing only. Not for clinical use.

⚠ CLINICAL SAFETY: Developer Mode bypasses all safety gate checks in the protocol wizard. It must never be active during patient treatment. Dev Mode does not persist across app restarts.

Activation

Action How
Toggle ON / OFF Hold Ctrl + Alt and click anywhere on the progress header bar (the horizontal bar with numbered step dots at the top of the wizard).
Visual Indicators: When active, a red DEV badge appears in the progress header, and the window title shows [DEV] at the end.

Features

1. Gate Bypass
All “Next” buttons are always enabled, regardless of checkbox state. You can freely advance through every wizard page without completing any checklist items. Checkboxes still function visually — they just don’t block navigation.
2. Page Jumper (Click Any Step Dot)
Every numbered step dot in the progress header becomes clickable. Click any dot to jump directly to that page — forward or backward — without sequential progression. The cursor changes to a pointer over clickable dots.
3. Readiness Value Overrides
An orange dashed 🔧 DEV OVERRIDES panel appears on the Autonomic Readiness page (Screen 6) with:
  • HR field — Type a heart rate value (BPM) to manually set the HR gauge.
  • RMSSD field — Type an RMSSD value (ms) to manually set the RMSSD gauge.
  • Force Baselines Locked button — Immediately marks the settling/baseline phases as complete, enabling the auto-verified checkboxes without waiting for the actual timer.

Typical Testing Workflow

  1. Launch VNS-TA.
  2. Ctrl + Alt + Click the progress header to activate Dev Mode.
  3. Confirm the red DEV badge appears and [DEV] is in the title bar.
  4. Click any step dot to jump to the page you want to test.
  5. On the Readiness page, use the override fields to set HR and RMSSD values that satisfy (or fail) the thresholds, and click “Force Baselines Locked” to bypass the timer.
  6. When done testing, Ctrl + Alt + Click the header again to deactivate.

Protocol Threshold Settings

These values are configurable via the gear icon (⚙) in the progress header, under the Protocol Thresholds section. Defaults are defined in config.py and user overrides persist in ~/.vns_ta_settings.json.

Setting Default Description
Max Resting HR 110 BPM Heart rate ceiling for readiness check.
Min RMSSD (GO) 45 ms Minimum baseline RMSSD to pass readiness. Below this, vagal tone is insufficient.
Critical RMSSD (NO-GO) 30 ms Below this, high sympathetic dominance is indicated. Triggers a clinical warning.
Min SpO₂ 95% Lower bound of acceptable oxygen saturation.
Max SpO₂ 100% Upper bound of acceptable oxygen saturation.
Note: The RMSSD thresholds may need clinical tuning per patient population. A healthy 16-year-old female has a median RMSSD of ~69 ms, but a medicated catatonic patient on clozapine will typically present significantly lower.
